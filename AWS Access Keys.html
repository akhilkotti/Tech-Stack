How to Create AWS Access Key & Secret Access Key

AWS Access Keys are used to make programmatic requests to AWS services from applications, CLI tools, SDKs, or CI/CD pipelines.

Step 1: Sign in to AWS Management Console

  • Go to https://aws.amazon.com/console
  • Sign in using your AWS root account or IAM user credentials

Step 2: Navigate to IAM (Identity and Access Management)

  • In the AWS Management Console, search for IAM in the search bar and click on it.
  • This will take you to the IAM Dashboard.

Step 3: Create an IAM User

  • On the IAM Dashboard, click on Users in the left-hand sidebar.
  • Click the Add user button.
  • Enter a user name (example: dev-access-user)
  • Under Select AWS access type, check the box for Programmatic access. This is necessary to generate an Access Key ID and Secret Access Key.
  • Click Next: Permissions.

Step 4: Assign Permissions

  1. Choose how you want to assign permissions to the user:
    • Existing policies:
      • Attach existing policies directly: Choose from the existing AWS managed policies.
      • Add user to group: Add the user to an existing group that has the appropriate policies.
      • Copy permissions from existing user: If you want to replicate permissions from another user.
      • Attach custom policies: If you have custom policies, you can attach them.
    • Add new policies:
      • AdministratorAccess (Full access)
      • AmazonS3FullAccess
      • AmazonEC2FullAccess
  2. Click Next: Tags to add optional tags, or click Next: Review to proceed.

Step 5: Review & Create User

  • Review username and permissions
  • Click Create user

Step 6: Download Access Key & Secret Key

After the user is created, AWS will display:

  • Access Key ID
  • Secret Access Key
Access Key ID: AKIAxxxxxxxxxxxx
Secret Access Key: xxxxxxxxxxxxxxxxxxxxx

Click Download .csv or copy the keys and store them securely.

⚠ Important Security Notes:
  • Secret Access Key is shown only once
  • Never commit keys to GitHub
  • Use IAM roles for EC2/Lambda when possible
  • It’s a good practice to rotate your keys periodically to enhance security. Make sure to update any applications or services that use the keys when you rotate them.

Optional: Configure AWS CLI

aws configure

Enter Access Key, Secret Key, region, and output format.
