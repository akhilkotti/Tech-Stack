üîπ O(n) in ArrayList

‚Üë Back to Features

Basics

  • ArrayList is backed by a resizable array
  • O(n) occurs during searching, removal by value, or insertion at index
  • Reason: elements must be traversed or shifted
list.contains(50);   // O(n)

Advanced

  • contains() checks elements one by one
  • remove(Object) = search + shift ‚Üí O(n)
  • add(index, element) shifts elements
Before: [10, 20, 30, 40]
remove(20)
After : [10, 30, 40]
Real-time impact: Searching large lists in REST APIs causes latency. 
Map<Id, Object>  // O(1) lookup instead of O(n)

üîπ O(n) in LinkedList

‚Üë Back to Features

Basics

  • Data stored as nodes (prev + next)
  • No random index access
  • Traversal required for access

Advanced

‚ùå Dangerous code (O(n¬≤))

    list.get(i);
}
‚úÖ Correct approach 
for(Integer val : list) {
    // O(n)
}

üîπ O(n) in HashSet

‚Üë Back to Features

Basics

  • Stores unique elements
  • No ordering or indexing
set.contains("Java"); // O(n) worst case

Advanced

  • Bad hashCode leads to bucket collision
  • Java 7: LinkedList ‚Üí O(n)
  • Java 8+: Red-Black Tree ‚Üí O(log n)
@Override
public int hashCode() {
    return 1;
}

üîπ O(n) in TreeSet

‚Üë Back to Features

Basics

  • Sorted set
  • Iteration requires visiting all nodes

Advanced

  • Red-Black Tree internally
  • Traversal always O(n)

üîπ O(n) in HashMap

‚Üë Back to Features

Basics

  • Key-value storage
  • Iteration over entries is O(n)

    // O(n)

Advanced

  • Worst case before Java 8: O(n)
  • Java 8+: treeified buckets ‚Üí O(log n)
